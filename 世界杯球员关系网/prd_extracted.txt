产品需求文档 (PRD)：2026 世界杯吃瓜星图 (MVP 版)
1. 产品概述
产品名称： 世界杯全景星图：核心球员与抓马关系网络 (World Cup Drama Star Map)
产品定位： 一款面向泛体育迷和伪球迷的 Web 可视化工具。以“八卦、矛盾、恩怨”为独特切入点，通过动态交互式关系网（力导向图），帮助用户在世界杯期间快速认清球队大名单、主教练及核心人物关系。
核心卖点： 真实人像节点、三行极简悬浮名片、点击连线看沉浸式“吃瓜长文”。
2. 目标用户与使用场景
目标用户： 赛前想要快速了解某支热门球队背景的泛球迷；喜欢体育八卦的社交媒体用户。
使用场景： 世界杯看球预热、社交媒体吃瓜求证、群聊分享装杯。
3. 功能需求详述
3.1 动态星空可视化网络 (核心展现层)
UI 风格： 暗色调（深灰/深蓝背景），以“星图”概念呈现，视觉极客且高级。
节点设计（Node）： * 全队覆盖：必须包含26人完整大名单球员 + 1名主教练。
视觉：全部使用真实、高清的头像照（圆形裁剪，带边框）。
层级区分：核心球员/主教练的头像边框加粗并高亮，边缘替补球员尺寸略小。
连线设计（Edge）： * 颜色映射机制：
* 红色连线： 负面/抓马关系（内讧、绯闻、世仇、更衣室矛盾）。
* 绿色连线： 正面/紧密关系（默契搭档、老乡、师徒情深）。
* 白色/灰色连线： 普通战术配合/位置竞争关系。
箭头指向：开启单向或双向箭头，展示关系的施加方向（如：主教练 -> 核心球员的“绝对信任”）。
3.2 节点悬浮卡片 (Tooltip - 极简信息流)
鼠标悬停在任何人物头像上时，半秒内弹出半透明黑色磨砂质感的悬浮框。严格限制为三行，拒绝冗长：
第一行（简介）： 极具网络梗或高度概括的评价（例：“比利时叹息之墙” / “更衣室定时炸弹”）。
第二行（薪资）： 满足大众好奇心的客观数据（例：周薪 £250,000 或 年薪 €1000万）。
第三行（现役）： 球员目前效力的俱乐部名称（例：皇家马德里）。
3.3 沉浸式吃瓜弹窗 (Drama Modal)
本产品的流量密码，触发条件为“点击两名球员之间的连线”。
标题： 带有情绪引导的粗体标题（如：🔥 惊天大瓜：友妻门）。
涉事人员： 明确标识争端/事件双方。
文风要求： 必须符合中国互联网用户的“吃瓜”阅读习惯。语言生动、带有叙事感、略带幽默或戏谑，避免死板的维基百科式翻译。
文案示例： “早在俱乐部时期，这两人就结下了梁子。A在采访中公开阴阳怪气B的防守形同虚设，导致B直接在更衣室摔了水瓶。两人长达三年在国家队集训时互不说话，全靠队长在中间当和事佬……”
3.4 底部信息联动盘 (Bottom Panel)
主角聚焦区（左侧）： 显示当前点击选中的球员/教练大头像、全名。
关系辐射区（右侧）： 以多列列表形式，罗列该主角的所有直接关系人。通过红、绿、白小圆点前缀，快速一览该主角在队内的人缘好坏。
4. 技术实现建议与数据结构
前端选型： HTML + CSS + JS 纯前端实现，核心依赖 Vis.js 库，直接挂载至 GitHub Pages，实现零成本部署与极速访问。
数据构建逻辑： 数据层与视图层完全分离。采用独立的 nodes.json 和 edges.json 存储球队结构。
自动化数据管线（拓展期规划）： 针对 48 支球队的庞大数据量，完全可以编写自动化脚本，定时抓取主流媒体（如人民日报体育版、光明日报等）的相关新闻与专栏文本，通过大模型 API 提炼出符合我们所需的 JSON 格式数据（包含人物属性、矛盾连线、吃瓜文案），大幅降低人工录入成本。
